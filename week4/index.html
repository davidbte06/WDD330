Chapter 8: Forms

Forms are a very common, we use it when we log in to our favorite sites for example.

  • We can create a form by using the tag form
  • we can access form elements: const form = document.form[0];
  • Forms have exclusive events: focus, blur, change
  • Of course one of the most important events is the submit this occurs when the form is submitted

    • Forms have controls for example:

  • input, select, textarea, button

    • We also have input fields, there are different types of input fields

  • Text: The default type of input field is text, which is used for entering a short piece of text, such as a username.
  • Password: is used to enter our passwords or secret information (input type='password')
  • Checkbox: (type='checkbox') are used to select different options (true) or (false)
  • Hidden Input Fields: (type='hidden') this type of input are not displayed by the browser but they have a value that contain information that is submitted when the form is submitted
  • File Input Fields: (type='file') are used to upload files, Most of the browser will provide a button to select a file from their file system
  • Other input fields: this are new inputs included in HTML5: number, tel, color
  • Text Areas: is used to enter long text such a comment or blog post
  • Buttons: there are different types of buttons, the default is submit

    • Chapter 12: Object Oriented Programming

    Object Oriented Programming (OOP) is a style of programming that involves separating the code into objects that have properties and methods, In object oriented programming, each object is capable of receiving messages, processing data and sending messages to other objects. Each object can be viewed as a small independent machine with a defined role or responsibility.

  • Encapsulation: is a protection or isolation mechanism for attributes and methods.
  • Polymorphism: is the ability of objects of a class to respond to the same message or event depending on the parameters used during its call.
  • Inheritance: Inheritance allows new classes to be defined based on existing classes in order to reuse code, thus generating a hierarchy of classes within an application.
